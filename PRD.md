# PRD — 逆波兰科学计算器（RPN Scientific Calculator）

## 1. 产品概述

构建一个支持完整科学运算的逆波兰表达式（Reverse Polish Notation）计算器。用户输入中缀表达式，系统将其转换为 AST（抽象语法树），再输出对应的逆波兰表达式，并将 AST 渲染为带标题的 JPG 图像。

---

## 2. 核心功能需求

### 2.1 输入 & 解析

| 功能 | 描述 |
|------|------|
| 词法分析 (Tokenizer) | 将输入字符串拆分为 Token 序列（数字、运算符、函数名、括号） |
| 语法分析 (Parser) | 将 Token 序列按优先级规则构建 AST |
| 优先级管理 (Operators) | 独立文件定义所有运算符优先级、结合性、元数 |

### 2.2 支持的运算符与函数

| 符号 | 类型 | 优先级 | 结合性 |
|------|------|--------|--------|
| `+` | 二元运算符 | 1 | 左 |
| `-` | 二元运算符 | 1 | 左 |
| `*` | 二元运算符 | 2 | 左 |
| `/` | 二元运算符 | 2 | 左 |
| `^` | 二元运算符 | 3 | 右 |
| `sin` | 一元函数 | 4 | 右 |
| `cos` | 一元函数 | 4 | 右 |
| `tan` | 一元函数 | 4 | 右 |
| `log` | 一元函数 | 4 | 右 |
| `( )` | 分组 | — | — |

### 2.3 RPN 转换

- 基于 Dijkstra 调度场算法（Shunting-yard）
- 从 AST 中序遍历生成后缀表达式
- 输出格式：空格分隔的 token 序列（例：`3 4 + 2 * sin`）

### 2.4 AST 可视化 & JPG 输出

每次运算完成后输出一张 JPG：

```
┌─────────────────────────────────────┐
│  sin(3 + 4) * 2                     │  ← 第一行：原始中缀表达式
│  RPN: 3 4 + sin 2 *                 │  ← 第二行：逆波兰表达式
├─────────────────────────────────────┤
│                                     │
│          [AST 树形图]               │
│                                     │
└─────────────────────────────────────┘
```

- 节点样式：运算符/函数节点用圆角矩形，数字节点用椭圆
- 边：从父节点到子节点，带方向
- 字体：清晰的等宽字体
- 背景：白色，有适当 padding
- 图片命名规则：`output/trees/<sanitized_expression>.jpg`

### 2.5 计算求值

- 基于 RPN 栈求值
- 支持浮点数
- 完整错误处理：除零、无效语法、未知函数

## 3. 技术要求

### 3.1 语言 & 依赖

| 组件 | 选型 |
|------|------|
| 语言 | Python 3.11+ |
| AST 可视化 | `graphviz` + `Pillow (PIL)` 或 `matplotlib` |
| 测试框架 | `pytest` + `pytest-cov` |
| 代码风格 | `black` + `flake8` |

### 3.2 开发方式

- **TDD（测试驱动开发）**：每个模块先写测试，再写实现
- **SDD（规格驱动开发）**：每个模块有明确的接口规格，测试即规格

---

## 4. 测试需求

### 4.1 覆盖维度

| 测试类型 | 说明 |
|----------|------|
| 单元测试 | 每个类/函数独立测试 |
| 集成测试 | 完整流程 infix → AST → RPN → result |
| 边界测试 | 负数、嵌套括号、连续函数、大数 |
| 异常测试 | 非法表达式、除零、括号不匹配 |

### 4.2 测试用例示例

| 中缀表达式 | 预期 RPN | 预期结果 |
|-----------|----------|---------|
| `3 + 4` | `3 4 +` | `7` |
| `2 ^ 3` | `2 3 ^` | `8` |
| `sin(0)` | `0 sin` | `0.0` |
| `(3 + 4) * 2` | `3 4 + 2 *` | `14` |
| `log(100) + 2` | `100 log 2 +` | `4.0` |
| `sin(3 + 4) * 2` | `3 4 + sin 2 *` | `~1.979` |
| `2 ^ (3 + 1)` | `2 3 1 + ^` | `16` |

## 5. 非功能性需求

- 每个模块文件不超过 200 行
- 所有公共函数有类型注解
- 测试覆盖率 ≥ 90%
- JPG 图像分辨率 ≥ 800×600
